GeoMap Catalog Manager

Comprehensive Documentation & Advanced Architectural Overview

1. Project Overview

The GeoMap Catalog Manager is a highly optimized desktop application designed to organize, visualize, and seamlessly stream massive geographic maps (up to 140-megapixels). It features an advanced asynchronous progressive rendering engine, strict hierarchical syncing to a local database, and Google Maps-style deep zoom tiling.

3. High-Level Architecture

The application employs an advanced Model-View-Controller (MVC) setup heavily optimized with background multi-threading to guarantee the UI never freezes during heavy computational tasks.

Frontend (UI & Threading)

  • GalleryPanel: Shows 300px cached thumbnails, sorts via regex numerical analysis, and filters by attributes.
  • SyncProgressDialog: A real-time monitoring UI connected to the SyncWorker thread, tracking ETA, file speeds, and tile generation.
  • MapViewerWindow: A borderless, hardware-accelerated full-screen viewer. Instantly flashes a 1920px preview while spawning an ImageLoaderThread to asynchronously stream the multi-gigabyte full-resolution image or LOD tiles.

Backend (Services)

  • FolderSyncService: Keeps the database perfectly synchronized with the physical maps/ filesystem. Dispatched to a background QThread.
  • CacheService: A robust asset pipeline utilizing Pillow (PIL). It systematically slices massive images into 256x256 multi-level Level-of-Detail (LOD) tile pyramids, identical to Google Maps.

4. Database Schema (SQLAlchemy)

  • Geographic Hierarchy: Country → State → District.
  • Map: Stores the original path and the cached paths (thumbnail_path, preview_path, has_tiles, tile_dir_path).
  • Tags & Metadata: Dynamic EAV (Entity-Attribute-Value) schema for completely customizable metadata fields.

5. Advanced Systems

Deep Zoom & Progressive Rendering Pipeline

Loading a 140-megapixel image instantly crashes standard image viewers. This application solves this via a three-tiered pipeline:

  1. Instant Preview: The MapViewerWindow instantly loads a pre-generated 1920px preview utilizing Qt.SmoothTransformation bilinear filtering.
  2. Asynchronous Threading: A background QThread leverages QImageReader (with an expanded 1GB C++ allocation limit) to decode the massive file without blocking the UI.
  3. Tile Streaming (LOD): If the user zooms, the TiledMapGraphicsItem dynamically calculates the exact mathematical viewport intersection and streams only the necessary 256x256 tiles from the hard drive, utilizing Qt's native 500MB QPixmapCache.

6. Tech Stack Summary

  • GUI & Threading: PySide6 (Qt6) - Native C++ wrapper
  • Database Engine: SQLite 3 with SQLAlchemy ORM
  • Asset Generation: Pillow (Python Imaging Library)
  • Visualization: Mermaid.js (Architecture Diagrams)
